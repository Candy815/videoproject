# 视频抄袭检测系统 - 安装说明

## 一、运行环境说明

### 1.1 系统要求
- **操作系统**: Windows 10/11, Linux, macOS
- **Python**: 3.8 或更高版本
- **内存**: 建议 8GB 以上
- **存储**: 至少 10GB 可用空间（用于视频处理和特征存储）

### 1.2 核心依赖服务
- **Milvus 2.3+**: 向量数据库（用于特征存储和相似性检索）
- **Docker**: 用于运行 Milvus 服务（可选，推荐使用）
- **FFmpeg**: 视频处理工具（需单独安装）

### 1.3 可选组件
- **GPU 支持**: CUDA 11.8+（用于加速特征提取）
- **Ollama**: 本地大模型服务（用于智能报告生成）
- **OpenAI API**: 云端大模型服务（替代 Ollama）

## 二、安装与部署步骤

### 2.1 环境准备
```bash
# 1. 安装 Python 3.8+（如未安装）
# 2. 安装 FFmpeg（视频处理必需）
# Windows: 下载并添加至 PATH
# Linux: sudo apt-get install ffmpeg
# macOS: brew install ffmpeg
```

### 2.2 启动 Milvus 数据库
```bash
# 使用 Docker 快速启动（推荐）
docker run -d --name milvus-standalone \
    -p 19530:19530 \
    -p 9091:9091 \
    milvusdb/milvus:v2.3.0

# 验证 Milvus 运行状态
docker ps | grep milvus
```

### 2.3 项目安装
```bash
# 1. 克隆项目（如未克隆）
# git clone <repository_url>
# cd videoproject

# 2. 创建虚拟环境
python -m venv venv

# 3. 激活虚拟环境
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate

# 4. 安装依赖（可能耗时 10-15 分钟）
pip install -r requirements.txt
```

### 2.4 配置环境变量
```bash
# 1. 复制环境配置模板
cp .env.example .env

# 2. 编辑 .env 文件，根据实际情况配置
# 主要配置项：
#   MILVUS_HOST=localhost
#   MILVUS_PORT=19530
#   LLM_PROVIDER=ollama（或 openai）
#   OLLAMA_BASE_URL=http://localhost:11434（如使用 Ollama）
#   OPENAI_API_KEY=your_key_here（如使用 OpenAI）
```

## 四、基本使用流程

### 4.1 启动 Web 界面（推荐）
```bash
# 在项目根目录下执行
python app.py
```
- 访问地址：http://localhost:7860
- 支持功能：文件上传、URL检测、参数配置、可视化结果

### 4.2 命令行使用
```bash
# 训练模式：构建视频特征库
python main.py train --input ./data/raw

# 检测模式：检测单个视频
python main.py detect --input ./data/raw/test.mp4

# URL 检测模式
python main.py detect --url "https://example.com/video.mp4"

# 批量检测
python main.py batch_detect --urls vsc_url_list.txt
```

### 4.3 Python API 调用
```python
from pipeline.detection_pipeline import DetectionPipeline

# 初始化检测流水线
pipeline = DetectionPipeline()

# 检测本地视频
report = pipeline.detect_infringement(
    video_path="path/to/video.mp4",
    similarity_threshold=0.7,
    top_k=10
)

# 检测在线视频
report = pipeline.detect_from_url(
    video_url="https://example.com/video.mp4",
    user_keywords=["侵权", "抄袭"],
    top_k=10
)
```

### 4.4 使用示例
1. **准备检测视频**：将视频文件放入 `data/raw/` 目录
2. **启动训练**：构建特征库（首次使用必需）
   ```bash
   python main.py train --input ./data/raw
   ```
3. **执行检测**：
   - Web界面：上传文件或输入URL，点击"开始检测"
   - 命令行：`python main.py detect --input ./data/raw/test.mp4`
4. **查看结果**：
   - Web界面：实时查看检测进度和结果
   - 报告文件：`data/reports/` 目录下的 HTML/JSON/TXT 文件

## 五、故障排除

### 5.1 常见问题
1. **Milvus 连接失败**
   - 检查 Milvus 服务状态：`docker ps | grep milvus`
   - 验证端口 19530 是否开放
   - 运行 `python check_milvus.py` 测试连接

2. **依赖安装失败**
   - 确保网络连接正常
   - 尝试使用国内镜像源：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
   - CLIP 安装较慢，请耐心等待

3. **视频处理错误**
   - 确认 FFmpeg 已正确安装：`ffmpeg -version`
   - 检查视频文件格式是否支持（MP4、AVI、MOV、MKV）

4. **内存不足**
   - 减少同时处理的视频数量
   - 修改 `config/settings.py` 中的 `MAX_FRAMES_PER_VIDEO`（默认 300）

### 5.2 获取帮助
- 查看详细文档：`README.md`
- 检查日志文件：`logs/system.log`
- 修改日志级别：在 `.env` 中设置 `LOG_LEVEL=DEBUG`

## 六、配置说明摘要

### 6.1 关键配置文件
- **`.env`**：环境变量（数据库连接、大模型配置）
- **`config/settings.py`**：系统参数（阈值、路径、超时设置）

### 6.2 重要参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `SIMILARITY_THRESHOLD` | 0.7 | 视觉特征相似度阈值 |
| `CLIP_SIMILARITY_THRESHOLD` | 0.6 | CLIP语义相似度阈值 |
| `MAX_FRAMES_PER_VIDEO` | 300 | 每个视频最大抽帧数 |
| `FRAME_EXTRACT_INTERVAL` | 1.0 | 抽帧间隔（秒） |

### 6.3 性能调优建议
1. **GPU 加速**：确保 PyTorch 支持 CUDA
2. **缓存启用**：`CLIP_USE_CACHE=True`（减少重复计算）
3. **分批处理**：大量视频建议分批训练
4. **存储清理**：定期清理 `data/frames/` 临时文件
